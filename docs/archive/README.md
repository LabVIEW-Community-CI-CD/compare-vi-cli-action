# Documentation Archive

This directory contains historical markdown retained for traceability and release archaeology.

## Guidance

- Treat archived documents as historical context, not the primary operator path.
- Prefer `README.md`, `AGENTS.md`, `docs/release/`, and current docs under `docs/` before consulting the archive.
- Archived material is intentionally grouped so future agents do not have to scan it to reach active guidance.

## Structure

- `releases/` - versioned release notes, archived PR descriptions, and archived release checklists
- `project-history/` - one-off project plans, implementation status documents, and historical remediation notes
